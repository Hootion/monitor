## 0.2.4 修复内容

### 地图和定位

- 高德定位采集改为更高精度配置。
- 前台位置上传节流从 12 秒缩短为 5 秒。
- Android 前台服务位置更新间隔缩短，最小距离改为 3 米。
- 地图默认缩放级别从 16 提高到 17.5。
- `AMapWidget` 增加手势识别配置，支持手指缩放、拖动和缩放范围。
- 位置坐标展示提高到 6 位小数。

### 应用使用异常

- Android 采集端跳过 Mutual Watch 自身包名：`com.mutualwatch.mutual_watch`。
- 单次应用使用会话最大按 4 小时处理。
- 单日屏幕时间最大按 24 小时处理。
- 客户端展示、NestJS 本地参考后端、Supabase API 都加入用量清洗逻辑。
- 线上历史数据删除 Mutual Watch 自身使用记录：1123 条。

### 导航和设置

- 底部导航改为：`总览 / 应用 / 记录 / 我的`。
- 移除底部栏里的 `绑定` 和 `隐私`。
- 新增 `我的` 页面。
- `绑定` 卡片移入 `我的`。
- `隐私`、权限、数据范围、账号操作移入 `我的 -> 设置`。

### 登录保持

- Access token 默认有效期从 15 分钟延长到 1 天。
- Refresh token 默认有效期从 30 天延长到 180 天。
- Refresh 时不再立即删除旧 refresh token，避免前台 App 和后台服务同时刷新导致竞态。
- 客户端刷新失败时会先重新读取本地 token，若后台服务已经刷新成功则复用新 token。
- 启动时如果 access token 不存在但 refresh token 存在，会先尝试刷新会话。

### 线上后端

- Supabase `api` Edge Function 部署为第 2 版。
- `api` 继续保持 `verify_jwt=false`，因为函数内部实现自定义鉴权。
- 线上异常使用数据已清理。

## 更新弹窗问题排查

线上验证结果：

- `api/health` 正常返回。
- `app-update?platform=android&currentVersionCode=2006` 正常返回新版。
- `app-update?platform=android&currentVersionCode=1` 正常返回新版。
- `0.2.4` APK 下载地址返回 200。

结论：

- “暂时连接不上服务器”表示手机网络层连接 API 失败，不等于服务器宕机。
- 可能原因包括 DNS 解析失败、连接超时、网络不可达、VPN/私有 DNS/代理或当前网络对 Supabase/Cloudflare 访问不稳定。
- 旧客户端的自动更新检查被登录状态挡住：未登录、登录过期或停留在登录页时不会检查更新。
- 自动更新检查原本挂在 `HomeShell`，只有进入主界面才会触发。

## 0.2.5 修复内容
