## 背景

用户反馈定位功能不满意，希望在总览页看到地图定位并实时更新；应用使用情况里希望显示真实应用名称，而不是难以识别的包名。后续又反馈“没有更新推送”和真机/模拟器打开地图闪退，因此今天同时完成了功能实现、发布链路补齐和闪退修复。

## 已完成

### 实时地图定位

- 在 Flutter 端新增高德地图和高德定位能力。
- 总览页新增“实时定位”地图面板，显示对方最新坐标、精度圈、更新时间、定位状态、手动刷新和回到定位点。
- 地图不可用时提供明确降级状态，包括无坐标、无权限、非 Android 平台、缺少高德 Key。
- 前台查看总览页时，每 15 秒刷新一次 `partner/overview`。
- 被观察端前台使用高德定位流采集位置，不再只依赖 `getLastKnownLocation`。
- Android 前台服务扩展为 `dataSync|location`，在登录、共享未暂停且权限满足时，后台尽力每 60 秒上传位置快照。

  - `FOREGROUND_SERVICE_LOCATION`
  - `QUERY_ALL_PACKAGES`
- 高德 Android Key 通过构建参数和 manifest placeholder 注入，不提交真实 Key。
- 本地 `.env.release.local` 已配置发布和高德 Key 相关变量，该文件被忽略，不进入仓库。

### 应用名称优化

- 应用使用会话和应用打开事件优先显示真实应用名称。

- 增加常见包名兜底名称。

## 闪退排查与修复

### 现象

- MuMu 模拟器打开地图后闪退。
- 后续 Android 真机也出现过一次打开地图闪退。

### 日志结论

真机 crash buffer 显示：

- 进程：`com.mutualwatch.mutual_watch`
- 线程：`GLThread`
- 信号：`SIGABRT`
- 原因：`JNI DETECTED ERROR IN APPLICATION: java_class == null`
- 崩溃库：`libAMapSDK_MAP_v8_1_0.so`

结论：闪退不是 Dart 层异常，也不是定位权限问题，而是旧版高德 3D 地图 native SDK 在当前 Android 16 真机环境中初始化 GL 地图时崩溃。

### 修复

- 将高德 3D 地图 SDK 从 `com.amap.api:3dmap:8.1.0` 升级到 `com.amap.api:3dmap:10.0.600`。
- 地图 Flutter 插件的 Android 编译依赖同步到 `3dmap:10.0.600`。
- 定位 Flutter 插件的 Android 编译依赖同步到 `location:6.4.9`。
- Release 构建开启 R8 和资源压缩。
- 新增 `mobile/android/app/proguard-rules.pro` 作为后续规则入口。

说明：曾尝试使用高德合包 `3dmap-location-search:10.1.200_loc6.4.9_sea9.7.4`，真机启动正常，但 APK 超过当前上传链路约 50 MiB 限制。最终改用独立 `3dmap:10.0.600`，既修复崩溃，也把 arm64 APK 控制在约 39.7 MiB。

## 后续发布：0.2.3

用户继续反馈滑到地图页面仍会闪退；重新查看 crash buffer 后确认：

- 真机已安装 `0.2.1`，但 release 包仍在 `GLThread` 上 `SIGABRT`。
- 崩溃原因仍是 `JNI DETECTED ERROR IN APPLICATION: java_class == null`，这次崩溃库为 `libAMapOpenMap.so`。
- `mobile/build/app/outputs/mapping/release/mapping.txt` 显示高德 SDK 关键类被 R8 混淆，native 库通过 JNI 按原 Java 类名查找时失败。
- MuMu x86_64 模拟器不会再 SIGABRT，但高德 native 报 `GLMapEngine.nativeMainThreadTrigger` 找不到实现，地图黑屏，属于 x86_64 模拟器 native 兼容问题。

修复：

- `mobile/android/app/proguard-rules.pro` 增加高德 SDK 和 Flutter 高德桥接的 keep/dontwarn 规则，避免 R8 混淆/裁剪高德 JNI 依赖类。
- `mobile/lib/main.dart` 增加 ABI 判断，只在 Android arm/arm64 和 iOS 上渲染 `AMapWidget`；x86/x64 模拟器显示降级提示，避免加载不兼容 native map。
- `scripts/publish_android_update.ps1` 改进上传响应校验，响应体缺少 `release` 时输出完整 JSON，便于后续排查。

验证：

- `flutter analyze` 通过。
- `flutter test` 通过，40 个测试全部成功。
- `flutter build apk --release --split-per-abi --build-name=0.2.3 --build-number=2006` 通过。
- 新 mapping 中关键高德类保持原名，例如 `GLMapEngine -> GLMapEngine`、`AMapNativeGlOverlayLayer -> AMapNativeGlOverlayLayer`。
- 真机安装本地 `0.2.2 / 2005` 测试包后滑到实时定位地图页，地图正常渲染，crash buffer 为空。
- MuMu 安装测试包后滑到实时定位区域，显示降级提示，crash buffer 为空。
